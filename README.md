# support-ticket-classification

## Опис
Telegram-бот для класифікації звернень користувачів (технічні, фінансові, інформаційні) з базовою статистикою моделі та логуванням дій.

## Реалізовані функції

### Основні можливості
- **Класифікація повідомлень**: Автоматична класифікація текстових звернень користувачів на 3 категорії (🛠️ - технічні, 💰 - фінансові, ℹ️ - інформаційні) за допомогою Naive Bayes класифікатора
- **Машинне навчання**: Тренування NLP-моделі на старті додатка з розбиттям даних на train/test набори (80/20 з фіксованим seed)
- **Метрики моделі**: Розрахунок та логування результатів тренування моделі (формування статистики для користувача)
### Команди та обробники
- `/start` - приєднання користувача та вивід доступних команд
- `/help` - виведення допомоги по користуванню ботом
- `/statistics` - показ метрик моделі класифікації
- `/history` - перегляд історії звернень користувача
- `Send request` - інтерактивна відправка звернення з класифікацією
- Обробка невідомих команд з повідомленням про помилку

### Взаємодія з користувачем
- Кнопка "Correct/Incorrect" для збору фідбеку про коректність класифікації
- Валідація текстових повідомлень (відкидання спецсимволів, однолітерних текстів)
- Відповідь на некоректні повідомлення (фото, файли, стікери, аудіо тощо)

### Система логування та зберігання
- Логування дій користувача та системних подій у файли (`user_actions.log`, `system_events.log`)
- Збереження фідбеку користувачів у `feedback.json`
- Обробка помилок Telegram API та мережевих збоїв з детальним логуванням

### Налаштування та конфігурація
- Конфігурація через змінні середовища (BOT_TOKEN)
- JSON-конфіг для базових параметрів
- Типо-безпечна реалізація (TypeScript)

## Основні модулі
- Бот та обробники команд/повідомлень: `module_1/src/bot`.
- Модель класифікації та навчальні дані: `module_1/src/nlp`.
- Утиліти (клавіатура, логер, shuffle): `module_1/src/utils`.
- Конфігурація: `module_1/src/configs`.
